
## Security

- I cloud provider come Azure, AWS e Google Cloud sono bersagli per gli hacker perchè sono milioni i clienti che usano i servizi. 
- I cloud provider sono sottoposti a regolari controlli di sicurezza da esperti e soddisfano i requisiti per le varie certificazioni di conformità internazionali. 
- Fornire ai clienti, gli strumenti per attivare e monitorare le applicazioni / data.

## Perchè ne abbiamo bisogno?

- La security è una challenge in IT, negli ultimi decenni
- Dobbiamo essere sicuri che il nostro cloud provider, non possa essere hackerato facilmente.

## Come si ottiene?
- Seguendo determinati standard ISO / internazionali per la sicurezza.
- Microsoft Security Response Center (MSRC)
- Always-on DDoS (Prevenire un attacco DDos)
- Azure Policy
- Role based access control (RBAC)
- Entra ID (Azure Active Directory)
- Versione aggiornata dei servizi
- Management degli aggiornamenti
- Crittografazione di default
- Funzioni di security come firewall
