

## Public Cloud

Il cloud pubblico è definito come servizi informatici offerti da fornitori terzi attraverso la rete Internet pubblica, rendendoli disponibili a chiunque voglia acquistarli.

 Azure possiede l'hardware della sua rete e della sua infrastruttura e lo mette a disposizione dell'utente o di chiunque abbia una carta di credito per affittare questi servizi.

Il 90% delle volte parliamo di Azure Public Cloud quando si parla di Azure.

## Private Cloud

Il cloud privato si basa in realtà su hardware e reti di proprietà privata, quindi il cloud privato è da intendersi come servizi di elaborazione offerti tramite Internet o una rete interna privata e solo a utenti selezionati invece del pubblico.

Quindi si può considerare un cloud privato, come quello governativo. Ha l'aspetto e il comportamento di un cloud, tranne che il proprietario dell'hardware è il cliente.

## Hybrid Cloud

L'hybrid cloud è un mix tra il private e il public Cloud.

Ad esempio il Database Stretch (funzione di SQL Server 2016), dove è possibile avere un DB locale che ha accesso al cloud quando ha bisogno di più spazio d'archiviazione aggiuntivo.
