
# Cosa sono le identità esterne?

- L'abilità di dare accesso sicuro alle tue risorse o applicazioni alle persone al di fuori della tua organizzazione
- Non c'è bisogno di creare o gestire un utente all'interno del proprio Active Directory


# Business-To-Business (B2B)

- B2B significa invitare partner, venditori o clienti nella tua organizzazione
- Loro sono dei guest
- Usano le loro credenziali per loggare
- Puoi assegnare ruoli o permessi, come se fossero degli utenti interni

B2B di solito è verso le aziende, consentire accesso a una risorsa ma non trattarli come dipendenti

# Business-To-Consumer (B2C)

- B2C significa invitare utenti o consumatori a usare le tue applicazioni
- Non appaiono nell'organizzazione
- Utenti che usano le "social identites" per loggare come Facebook, Google, Linkedin o Microsoft


# B2B vs B2C

- B2B è la collaborazione tra aziende, dando accesso guest al tenant
- B2C è per l'utilizzo di applicazioni per utenti

# Benefit della Sicurezza

- **Single Sign-on (SSO)**: Non c'è bisogno di gestione di una password
- **Multi-Factor Authentication (MFA)**: per una sicurezza maggiore
-  Puoi usare Entra ID feature come l'accesso condizionale
