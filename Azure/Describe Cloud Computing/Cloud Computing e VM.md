# VM 

Con Azure, si ha la possibilità di creare una VM in un data center specifico, in questo modo gestiamo la VM in più Data Center per evitare casi di stacco corrente etc.. 

Cosi facendo possiamo creare la macchina con la ridondanza. 

## Sistemi di SO

Si può anche inserire la versione del SO come:
-  Windows
- Linux
- Red Hat
E si sceglie anche l'architettura che può essere: 

- Arm64 
    
- X64 
    

Possiamo scegliere CPU, RAM, Memoria con ognuna la propria funzione. 

Bisogna ricordarsi di selezionare RDP come porta per collegarsi in remoto e ad aprire le varie porte che possono essere: 

- RDP: 3389 
    
- HTTPS: 443 
    
- HTTP: 80 
    
- SSH: 22


## RETE VIRTUALE

E' possibile inserire la VM in una rete virtuale esistente o già creandone una.


## MANAGEMENT

Possiamo inserire Backup con annesse le policy, Auto-Shutdown, Antivirus e gestione Windows Updates.

La sezione dedicata ai log, si possono mettere le sezioni per configurare la macchina con i log all'avvio di windows.
